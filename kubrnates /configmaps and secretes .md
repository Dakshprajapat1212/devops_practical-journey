## ЁЯз╛ ConfigMap vs Secret тАФ Kya hota hai?

| Resource      | Matlab kya hai? |
|---------------|------------------|
| **ConfigMap** | Non-sensitive config values store karta hai тАФ jaise app settings, env vars |
| **Secret**    | Sensitive data store karta hai тАФ jaise passwords, tokens, API keys |

Dono ka kaam hai: **container ke andar config values dena** тАФ lekin ek public hai (ConfigMap), ek private (Secret).

---

## ЁЯФз ConfigMap тАФ Asaan Example

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: my-config
data:
  DB_HOST: localhost
  DB_PORT: "5432"
```

Tu isko Pod mein inject kar sakta hai:
- As **environment variable**
- As **mounted file**

---

## ЁЯФР Secret тАФ Asaan Example

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: my-secret
type: Opaque
data:
  DB_PASSWORD: cGFzc3dvcmQ=   # base64 encoded "password"
```

Tu isko bhi Pod mein inject kar sakta hai тАФ lekin ye **base64 encoded hota hai**.  
Security ke liye, Secret ko RBAC se protect karna padta hai.

---

## ЁЯза Real-Life Analogy

Soch le tu ek restaurant chala raha hai:

- **ConfigMap** = Menu board pe likha hua: тАЬChai тВ╣10, Coffee тВ╣20тАЭ тЖТ Public info
- **Secret** = Locker mein rakha cash box ka password тЖТ Private info

## ЁЯЫая╕П Access kaise hota hai?

1. **Environment Variable**:
   ```yaml
   envFrom:
     - configMapRef:
         name: my-config
     - secretRef:
         name: my-secret
   ```

2. **Volume Mount**:
   ```yaml
   volumeMounts:
     - name: config-vol
       mountPath: /etc/config
   volumes:
     - name: config-vol
       configMap:
         name: my-config
   ```

3. **CLI se decode**:
   ```bash
   echo cGFzc3dvcmQ= | base64 -d
   ```

---

## тЪая╕П Important Tips

- Kabhi bhi **password ConfigMap mein mat daalna** тАФ use Secret.
- Secret sirf **base64 encoded** hota hai тАФ encrypted nahi, jab tak tu тАЬencryption at restтАЭ enable na kare.
- Tu Secret ko **RBAC se restrict** kar sakta hai тАФ taaki koi unauthorized user na access kare.
